# multi_game_portal

## 見てほしいポイント
- モンスター詳細ページ（UI構造）
- ログイン機能（UX）

## 概要
Djangoを用いて開発したゲーム攻略サイトです。  
モンスター、アイテム、マップ、ボス攻略など複数カテゴリの情報を扱い、ユーザーが情報を探しやすいUIを意識して設計しました。

## 使用技術
- Python / Django
- JavaScript
- CSS
- HTML

## 主な機能
- ログイン / 新規登録 / パスワードリセット
- コメント機能
- モンスター、アイテム、マップ情報の閲覧
- サイドバー目次とアンカーリンクによるページ内移動
- 画像クリックによるモーダル表示
- ホバーによる詳細表示

## 工夫した点
- サイドバー目次とアンカーリンクにより、情報量の多いページでも移動しやすくした
- モーダル表示やホバー表示により、情報を段階的に提示
- 各ページの情報構造を統一し、閲覧しやすさを向上

## 課題
- HTMLやCSSの重複が多く、保守性に課題があった
- テンプレート設計や共通化が不十分だった

## 学んだこと
- UIの見た目だけでなく、再利用性や構造設計が重要であること
- ページ数が増えるほど設計の重要性が増すこと

## 今後改善したい点
- 共通コンポーネント化
- CSS設計の整理
- 再利用性の高い構造への改善
